# ClassFlow MVP 범위 정의 (v1.0)

## 1. 문서 목적
본 문서는 요구사항 초안 v1.0을 실제 개발 가능한 MVP 범위로 고정한다.

## 2. 범위 원칙
- 토론 세션 운영 완결성 우선: 생성 -> 입장 -> 진행 -> 기록 -> 승인.
- Station 입력 최소화, Teacher 통제/승인 중심.
- 세특은 추천 자동화 + 최종 교사 선택.
- STT/고급 AI 생성문은 MVP에서 제외하고 이벤트 기반 요약만 제공.

## 3. MVP 포함 범위 (Must)

### 3.1 Teacher
- 대시보드
  - 빠른 시작: 새 토론 세션, 새 발표 세션.
  - 최근 세션 목록(최대 10건).
  - 최근 세특 작업(Primary/Alt 표시).
- 세션 목록
  - 토론/발표 통합 목록.
  - 필터: 기간, 반, 유형, 상태(시작 전/진행 중/종료).
- 토론 세션 생성
  - 필수: 활동 유형, 학생 목록(전체 기본 선택), 조 편성, 찬반 배정(수동/랜덤), 슬롯 템플릿, 슬롯별 시간, 라운드 수, 기록시간(기본 60초).
  - 선택: 주제, 슬롯 담당자 지정(수동/자동 로테이션).
- 토론 세션 상세
  - 시작 전: 조/찬반/슬롯 담당자 수정, 로비 입장 현황, 초대 발송.
  - 진행 중: 현재 라운드/슬롯/타이머, 조별 제출 상태, 일시정지/재개/종료, 긴급 슬롯 담당자 변경.
  - 종료 후: 조별 요약, 전체 승인 1탭, 제한적 무효 처리.
- 발표 세션
  - 생성: 주제(선택), 발표자 선택, 순서 편집(선택).
  - 진행: 이전/다음 수동 이동, 동의 학생만 시작.
  - 종료: 완료/미완료/미동의완료 집계.
- 학생/세특 모드
  - 학생 타임라인(세션 블록 최신순).
  - Primary + Alt 묶음 관리.
  - Primary 자동 추천 3~5건.
  - 교사 수정/확정.

### 3.2 Debate Station
- 초기 등록: 교사 승인 후 역할(교탁/조) 부여.
- 대기 화면: 역할 표시, 초대 카드, 입장.
- 토론 입장 후
  - 공통: 조 인원, 찬반, 슬롯 순서, 현재 슬롯 하이라이트.
  - 교탁: 라운드/슬롯/타이머 + 조별 제출 상태.
  - 조 스테이션: 라운드/슬롯/타이머, 라운드 종료 후 60초 기록 입력.
  - 기록 항목: 발언자, 발언 유형, 논리 전략(8개), 저장.
  - 저장 후 3초 되돌리기.
  - 기록시간 종료 시 자동 잠금.

## 4. MVP 제외 범위 (Later)
- STT 자동 전사 파이프라인.
- AI 서술문 자동 생성/문체 변환.
- 고급 분석 대시보드(통계 차트).
- 다중 학급 동시 생중계 운영.
- 외부 LMS 연동.

## 5. 정책 고정 (미확정 항목 확정)

### 5.1 토론 시작 조건
- 시작 버튼 활성화 조건:
  - 교탁 스테이션 1대 입장 완료.
  - 모든 활성 조(`team.is_active=true`) 스테이션 최소 1대씩 입장 완료.
- 교사 강제 시작 허용:
  - 위 조건 미충족이어도 `force_start=true`로 시작 가능.
  - 감사 로그(`audit_log`)에 사유 저장.

### 5.2 RecordEvent 저장 제한
- 라운드당 조별 최대 `N=6`건.
- 저장 직후 3초 내 작성자만 되돌리기 가능.
- 잠금 후 수정:
  - 교사만 `unlock_window_sec=120` 범위에서 재오픈 가능.
  - 재오픈 이력 로그 필수.

### 5.3 초대/입장 정책
- 초대 만료: 발급 후 24시간.
- 중복 입장:
  - 같은 스테이션 ID 재입장은 기존 세션 갱신(중복 생성 없음).
  - 같은 역할로 다른 기기 입장 시 최근 heartbeat 기기만 활성, 기존 기기는 읽기전용 전환.
- 오입장:
  - 조 스테이션은 교사가 즉시 재배정 가능.
  - 재배정 전 기록은 원 역할 기준으로 보존.

### 5.4 오프라인/재전송 정책
- 조 스테이션은 `local queue`에 이벤트 단위 임시 저장.
- 재전송: 2초, 5초, 10초, 20초 지수 백오프 후 20초 고정 재시도.
- 충돌 규칙:
  - 동일 `event_client_id`는 서버 멱등 처리.
  - 상충 수정은 서버 최종 시간 기준 + 교사 수정 우선.

### 5.5 세특 번들 자동 선택 규칙
- Primary 3~5건 자동 선택 기준:
  - 최근성 40%, 발언 유형 다양성 30%, 전략 태그 다양성 30%.
- 동일 라운드/동일 태그 중복은 감점.
- Alt는 Primary 제외 상위 후보를 순차 생성.

### 5.6 발표 완료 기준
- `completed`: 동의 + 실제 발표 완료.
- `incomplete`: 동의했으나 발표 미완료.
- `consent_skipped`: 미동의로 녹음 제외, 발표 상태 집계는 별도.

### 5.7 권한 모델 (MVP)
- Teacher:
  - 세션 생성/수정/시작/일시정지/종료/강제시작/재오픈/전체승인/무효처리.
- Desk Station:
  - 진행 모니터링(읽기 중심), 직접 수정 불가.
- Team Station:
  - 본인 팀 RecordEvent 작성/3초 되돌리기.
  - 잠금 해제/무효화 불가.

## 6. 수용 기준 (측정 조건 포함)
- AC-01: 교사는 20초 내 토론 세션 생성 가능.
  - 측정 조건: 반 30명, 기본 슬롯 템플릿 3개, 랜덤 찬반, 기본 설정값.
- AC-02: 모든 조 스테이션 입장 후 시작 버튼 자동 활성화.
- AC-03: 진행 중 조별 현재 슬롯(찬반/유형/학생) 실시간 반영(지연 1초 이내 목표).
- AC-04: 라운드 종료 후 60초 입력창 활성, 종료 즉시 잠금.
- AC-05: 종료 후 전체 승인 1탭 처리.
- AC-06: 세특 모드 진입 시 Primary 3~5건 자동 선택 표시.

## 7. 릴리스 컷
- Release A (운영 가능 MVP): 3장(Teacher 토론, Station 토론, 승인/세특 Primary).
- Release B (보완): 발표 세션 완성도 개선, Alt 추천 고도화.
