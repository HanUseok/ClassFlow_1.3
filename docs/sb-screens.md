# 화면정의서 (페이지 단위)

## 기준
- 라우트 단위(`app/**/page.tsx`) 전수
- 사용자에게 보이는 UI/행동/상태 중심
- 구현 세부(API/DB/컴포넌트 내부 구조) 제외

---

## 1) `/`
- 화면 목적: 루트 진입을 Teacher 메인으로 즉시 연결.
- 화면 구성: 실제 표시 화면 없음(리다이렉트 전용).
- 주요 컴포넌트/섹션: 없음.
- 표시 데이터: 없음.
- 사용자 액션: 없음.
- 상태별 UI: 없음.
- 네비게이션: `/` → 자동 `/teacher`.
- 동적 라우트 예외: 해당 없음.

## 2) `/teacher`
- 화면 목적: Teacher 홈(빠른 시작/대기 세션/최근 세션 접근 허브).
- 화면 구성: 상단 네비(공통), 좌측 빠른 시작, 우측 콘텐츠(대기 세션/최근 세션/최근 활동).
- 주요 컴포넌트/섹션:
  - Quick Start 버튼 2개(토론/발표 생성)
  - Pending 세션 카드 리스트
  - Recent 세션 카드(최대 4)
  - Recent Activity 빈 상태
- 표시 데이터:
  - 세션 제목, 주제, 학급명, 날짜, 상태
  - recent/pending 여부
- 사용자 액션:
  - 토론/발표 생성 이동
  - 대기 세션 `수정`, `진입하기`
  - 최근 세션 카드 클릭
  - 전체보기 클릭
- 상태별 UI:
  - 로딩: "세션 불러오는 중..."
  - 빈 상태: 대기세션/최근세션/최근활동 각각 빈 상태 메시지
  - 에러/권한: 명시 없음
- 네비게이션:
  - 생성: `/teacher/sessions/create?type=...`
  - 상세: `/teacher/sessions/{id}`
  - 목록: `/teacher/sessions`
- 동적 라우트 예외: 없음.

## 3) `/teacher/sessions`
- 화면 목적: 세션 전체 관리(탐색/삭제/상태별 진입).
- 화면 구성:
  - 상단 헤더(제목 + 전체삭제 + 생성 버튼 2개)
  - 필터 바(유형/상태/검색)
  - 세션 리스트 카드
- 주요 컴포넌트/섹션:
  - 상태 라벨/색상
  - 상태별 메인 액션 버튼
  - 삭제 버튼
- 표시 데이터:
  - 제목, 주제(없으면 "주제 없음"), 날짜, 상태
  - 유형/상태 필터 값, 검색어
- 사용자 액션:
  - 유형/상태 필터 변경
  - 검색 입력
  - 전체삭제(confirm)
  - 세션별 삭제(confirm)
  - 상태별 메인 액션 클릭
- 상태별 UI:
  - 로딩: hydrated 전 로딩 메시지
  - 빈 상태: "조건에 맞는 세션이 없습니다."
  - 실패: confirm 취소 시 변화 없음
- 네비게이션:
  - Pending: `/teacher/sessions/create?sessionId=...`
  - Live: `/teacher/sessions/{id}`
  - Ended: `/teacher/sessions/{id}/report`
  - 생성 버튼: create page
- 동적 라우트 예외: 없음.

## 4) `/teacher/sessions/create`
- 화면 목적: 토론/발표 세션 생성 또는 기존 세션 수정.
- 화면 구성:
  - 상단 Back + 타이틀
  - 공통 입력(주제)
  - 토론 Step 탭(세션설정/조배치/논거카드) 또는 발표설정 영역
  - 하단 저장/세션실행 버튼
- 주요 컴포넌트/섹션:
  - DebateRosterStep
  - Debate 배치 패널(미배치/조별 슬롯/랜덤배정)
  - DebateCardsStep
  - PresentationConfigStep
- 표시 데이터:
  - 학급/학생 목록
  - 토론 모드/진행 방식/단계 설정
  - 슬롯 수/배치 현황
  - 논거 카드 목록(진영/제목/주장)
  - 발표자 순서/녹음 여부/발표시간
- 사용자 액션:
  - 뒤로가기(router.back)
  - 모든 폼 입력/선택/토글
  - 드래그앤드롭 배치
  - 랜덤배정 다이얼로그 조작
  - 논거카드 추가/삭제/재생성
  - 저장/세션실행
- 상태별 UI:
  - Suspense fallback 로딩 문구
  - Debate cards 생성 중 로딩 패널
  - 유효성 미충족 시 버튼 disabled + 경고 문구
  - 수정 모드면 기존 데이터 초기 채움
- 네비게이션:
  - 저장: `/teacher/sessions`
  - 세션실행: `/teacher/sessions/{id}`
  - 백: 이전 페이지
- 동적 라우트 예외:
  - `sessionId`가 유효하지 않으면 신규 생성처럼 동작(명시적 에러 UI 없음).

## 5) `/teacher/sessions/[id]`
- 화면 목적: 단일 세션 진행/제어(토론/발표 공통 진입점).
- 화면 구성:
  - 상단 뒤로가기 + 제목/주제
  - (토론 teacher-guided일 때) 진행/관리 토글 + 배지
  - 본문: Debate 또는 Presentation 분기
- 주요 컴포넌트/섹션:
  - Debate: LiveDebateScreen, QuickAddScreen, DebateGroupPanel
  - Presentation: PresentationView
- 표시 데이터:
  - 세션 기본정보(제목/주제/상태)
  - 조별 구성, 발언자/단계, 토론 모드
  - 발표자/타이머/녹음 여부
- 사용자 액션:
  - 토론 시작/종료/일시정지/재개
  - 발언자 이전/다음/선택
  - 단계 변경, 발언순서 재정렬
  - 발표 시작/종료/다음 발표자
- 상태별 UI:
  - 로딩: "세션 불러오는 중..."
  - 세션 미존재: "세션을 찾을 수 없습니다." + 목록복귀
  - 토론 종료(`Ended`): 관리 화면 읽기전용 안내
  - 1조 데이터 없음 안내(teacher guided progress 모드)
- 네비게이션:
  - 뒤로: `/teacher/sessions`
  - 토론 종료: `/station/report?...`
- 동적 라우트 예외:
  - id 미존재/잘못됨 시 상세 조작 불가, 안내 화면으로 대체.

## 6) `/teacher/sessions/[id]/report`
- 화면 목적: 교사용 세션 리포트 조회.
- 화면 구성:
  - 뒤로가기
  - 제목/주제 헤더
  - 전체 학생 카드
  - 리포트 테이블(자유토론/순서토론 분기)
- 주요 컴포넌트/섹션:
  - 자유토론 4열 매핑 테이블
  - 순서토론 단계별 매핑 테이블
- 표시 데이터:
  - 세션 제목/주제
  - 토론 멤버 이름 목록
  - 논거/사고/키워드 문자열
- 사용자 액션:
  - 목록 복귀 링크 클릭
- 상태별 UI:
  - 로딩: "리포트 불러오는 중..."
  - 예외: 세션 없음/토론 아님 안내 + 복귀 링크
  - 빈 멤버: "학생 정보 없음"
- 네비게이션:
  - `/teacher/sessions`
- 동적 라우트 예외:
  - 잘못된 id이면 리포트 대신 예외 카드 표시.

## 7) `/teacher/students`
- 화면 목적: 학생 목록 탐색과 상세 진입.
- 화면 구성:
  - 제목
  - 검색 입력
  - 학급 필터 Select
  - 학생 카드 그리드
- 주요 컴포넌트/섹션:
  - 검색 아이콘 + Input
  - 학급 Select
  - 학생 카드(이름/학급)
- 표시 데이터:
  - 학생 이름, 학급명
  - 필터 선택값
- 사용자 액션:
  - 검색 입력
  - 학급 필터 선택
  - 학생 카드 클릭
- 상태별 UI:
  - 결과 없음: "학생이 없습니다."
  - 로딩/에러 UI 명시 없음
- 네비게이션:
  - `/teacher/students/{id}`
- 동적 라우트 예외: 없음.

## 8) `/teacher/students/[id]`
- 화면 목적: 학생 개인 프로필 + 세션 참여/발언 이력 조회.
- 화면 구성:
  - 목록 복귀 링크
  - 학생 헤더(아바타/이름/학급)
  - StudentSessions(타입 필터 + 세션 아코디언)
- 주요 컴포넌트/섹션:
  - 타입 필터 칩(all/debate/presentation)
  - 세션 행(접기/펼치기)
  - 이벤트 리스트(발언 유형/메모/라운드/팀)
- 표시 데이터:
  - 학생 정보
  - 참여 세션 리스트
  - 이벤트 note, round, team, speechType
- 사용자 액션:
  - 타입 필터 클릭
  - 세션 아코디언 열기/닫기
  - 목록으로 링크 클릭
- 상태별 UI:
  - 학생 없음: `notFound()` (프레임워크 404)
  - 세션 없음: 빈 상태 메시지
  - 세션은 있으나 이벤트 없음: 세션 내부 안내 문구
- 네비게이션:
  - `/teacher/students`
- 동적 라우트 예외:
  - id가 잘못되면 notFound 처리.

## 9) `/teacher/settings`
- 화면 목적: 명단 관리/스테이션 관리 UI 제공.
- 화면 구성:
  - 제목
  - Tabs(`roster`, `stations`)
  - 탭별 패널
- 주요 컴포넌트/섹션:
  - RosterManagement: 학급 카드/학생 리스트/CSV 업로드 버튼
  - StationManagement: 스테이션 목록/상태/역할 Select/삭제
- 표시 데이터:
  - 학급명, 학생수, 학생명/id
  - 스테이션명, 연결상태, 역할
- 사용자 액션:
  - 탭 전환
  - 스테이션 역할 변경
  - 스테이션 삭제
  - CSV 업로드 버튼 클릭(UI)
- 상태별 UI:
  - 스테이션 목록 비면 빈 상태
  - 로딩/에러 별도 없음
- 네비게이션:
  - 페이지 내 탭 전환(라우트 변경 없음)
- 동적 라우트 예외: 없음.

## 10) `/station`
- 화면 목적: 스테이션 참여자가 토론 대기/진행/종료를 수행.
- 화면 구성:
  - 활성 토론 없을 때 안내 패널
  - waiting 상태: WaitingScreen
  - live 상태: LiveDebateScreen + QuickAddScreen
- 주요 컴포넌트/섹션:
  - waiting 정보카드(주제/교사/생성시각/인원/순서유무)
  - live 토론 패널 + 기록 패널
- 표시 데이터:
  - 활성 토론 세션 정보
  - 발언자 목록/현재 발언자/단계/타이머
- 사용자 액션:
  - 입장
  - 발언 제어/순서 변경/단계 변경
  - 카드 장착/프리뷰/기록 저장
  - 토론 종료
- 상태별 UI:
  - 로딩: "스테이션 불러오는 중..."
  - 활성 토론 없음: 안내 메시지
  - waiting/live 내부 상태 분기
- 네비게이션:
  - 토론 종료 시 `/station/report?...&source=station`
- 동적 라우트 예외:
  - 동적 세그먼트 없음.

## 11) `/station/report`
- 화면 목적: 토론 결과 리포트 조회 및(조건부) 관리 화면 전환.
- 화면 구성:
  - (source가 station이 아닐 때) 세션 헤더 + view 토글 + 배지
  - 본문: report view 또는 manage view
  - report view: 자유/순서 토론 테이블
  - manage view: ReportManageView
- 주요 컴포넌트/섹션:
  - SessionTopHeader
  - report/manage 토글 버튼
  - 발표자 매핑 테이블
  - Station 복귀 버튼(조건부)
- 표시 데이터:
  - round, phase, logs, names
  - sessionId, sessionTitle, sessionStatus
  - teacherGuided, groupCount, groupLayout
- 사용자 액션:
  - view 전환 클릭(report/manage)
  - 세션목록/스테이션 복귀 클릭
- 상태별 UI:
  - 자유토론 로그 없음 / 순서토론 로그 없음 메시지
  - logs/groupLayout JSON 파싱 실패 시 fallback 표시
  - `source=station`이면 헤더 축소 + report view 강제
- 네비게이션:
  - `/teacher/sessions` (teacher 출처)
  - `/station` (station 출처 또는 sessionId 없음)
  - 동일 라우트 query 전환(`view`)
- 동적 라우트 예외:
  - 동적 세그먼트는 없으나 query 누락/오류를 fallback으로 흡수.
